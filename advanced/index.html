These optional features add more power to Getopt in exchange for some additional configuration work.

One thing all of these have in common is that they make use of chainable methods. That makes it easy to add any number of them to an Option object, for example: {% highlight php startinline %} $getopt = new Getopt(array( (new Option('o', 'option', Getopt::REQUIRED_ARGUMENT)) ->setDescription('Description of option') ->setDefaultValue('default') )); {% endhighlight %}

Note that chaining methods directly off the constructor (new Option(...)->set...) only works in PHP 5.4 and above. As of Getopt 2.2, users restricted to PHP 5.3 can use the static create() method instead:

{% highlight php startinline %} $getopt = new Getopt(array( Option::create('o', 'option', Getopt::REQUIRED_ARGUMENT) ->setDescription('Description of option') ->setDefaultValue('default') )); {% endhighlight %}
